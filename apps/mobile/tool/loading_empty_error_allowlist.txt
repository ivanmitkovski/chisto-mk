# Documented exemptions for the loading/empty/error-triple heuristic.
#
# This guard scans screens for a recognizable loading + empty + error state
# triple. The screens below are intentionally exempt because either:
#  - they are static/form/flow screens with no async list to page through
#    (auth onboarding/otp/forgot-password/sign-up/splash/location gate,
#    profile settings forms, organizer toolkit/quiz flows, attendee QR scanner), or
#  - they render async state through shared widgets the heuristic does not
#    recognize (AppErrorView / skeletons / AppRefreshIndicator inside child
#    widgets rather than the screen body).
# The ratchet only allows this list to shrink; new screens must ship the triple.
